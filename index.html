Linter Rules ¶

This document is automatically published on github pages. To update it edit docs/rules.md in the github repo.

  • ✅ means the rule has been ported/implemented
  • ❎ means the rule has been removed
  • ❌ means the rule hasn't been ported yet

A ✅ next to a section of rules means they have all been filed in our issues database.
